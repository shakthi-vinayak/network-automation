The repository is a flat, multi-tool orchestration surface rather than a single application: Ansible (inventories/group_vars/host_vars/playbooks/roles) drives device configuration through Jinja2 templates under `templates/<vendor>/`; Python modules in `python/` provide reusable clients (NETCONF, RESTCONF, SSH, SNMP, telemetry, config generation, validation, backup, compliance); `bots/` expose REST APIs plus ChatOps integrations for self-service operations; `terraform/`, `packer/`, and `policies/` cover cloud IaC, image builds, and OPA/Sentinel policy-as-code; `.github/workflows/` defines the GitOps pipeline (lint → schema + secrets scan → unit/Molecule/Batfish tests → dry run → approval → deploy → verify). Dependency direction is one-way: bots call Python modules, which call Ansible roles/templates against devices; monitoring (`monitoring/prometheus|grafana|otel`) and compliance (`compliance/`) observe outcomes but do not feed back into deployment. Secrets are abstracted behind an adapter layer supporting HashiCorp Vault, AWS Secrets Manager, Azure Key Vault, CyberArk, and Ansible Vault, with OIDC federation for CI/CD.
